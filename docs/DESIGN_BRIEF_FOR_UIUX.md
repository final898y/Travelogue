## 專案概述

### 什麼是 Travelogue？

Travelogue 是一款**溫馨自然風格**的旅遊規劃 Web App，幫助使用者輕鬆規劃旅程、管理預訂、記錄支出、撰寫旅行日誌。

### 核心價值主張

**「像寫手帳一樣規劃旅行」**

- 🌿 溫暖療癒的視覺體驗
- ✨ 簡單直覺的操作流程
- 📱 隨時隨地都能使用（手機優先）
- 🔗 輕鬆分享給旅伴

### 目標使用者

#### 主要使用者（Primary）

- **年齡**: 25-40 歲
- **特徵**: 喜歡規劃旅行、注重細節、享受記錄過程
- **行為**: 會提前做功課、喜歡有條理地安排行程
- **裝置**: 主要使用手機，偶爾用平板或電腦

#### 次要使用者（Secondary）

- **情侶/家庭旅行規劃者**: 需要協調多人行程
- **背包客**: 重視預算控制與分攤計算
- **旅遊愛好者**: 喜歡記錄旅行回憶

### 使用場景

| 階段       | 場景                           | 使用需求                     |
| ---------- | ------------------------------ | ---------------------------- |
| **規劃期** | 坐在咖啡廳規劃下個月的京都之旅 | 新增行程、查詢景點、預訂飯店 |
| **旅行中** | 在機場候機時確認住宿資訊       | 快速查看預訂細節、地圖導航   |
| **旅行中** | 在餐廳結帳後記錄花費           | 快速記帳、自動換算匯率       |
| **旅行後** | 回到家整理照片與心得           | 上傳照片、撰寫日誌           |

## 設計理念與風格

### 視覺關鍵字

```
溫馨 Cozy  |  手帳 Journal  |  自然 Nature  |  療癒 Healing
圓潤 Rounded  |  柔和 Soft  |  輕盈 Light  |  友善 Friendly
```

### 設計靈感來源

#### ✅ 參考方向

- **手帳文化**: 日本文具品牌（如 Hobonichi、Midori）的排版與配色
- **自然系美學**: 植物、大地、森林的色調
- **Notion**: 清晰的資訊層級與卡片式佈局
- **Bear Notes**: 簡約溫暖的介面設計
- **Google Calendar**: 直覺的時間軸呈現

#### ❌ 避免的風格

- 過於鮮豔或對比強烈的色彩
- 冷硬的商務風格
- 過度裝飾的 Skeuomorphism（擬物化）
- 極簡到缺乏溫度的設計

### 設計原則

#### 1. 溫暖友善 (Warm & Friendly)

- 使用圓角而非銳角
- 柔和的陰影取代硬陰影
- 溫暖的配色（米色、綠色、大地色）
- 親切的文案語氣

#### 2. 簡約清晰 (Simple & Clear)

- 每個畫面聚焦一個主要任務
- 資訊層級分明（標題 > 內容 > 輔助資訊）
- 減少不必要的裝飾
- 留白適當，不壓迫

#### 3. 觸感回饋 (Tactile Feedback)

- 按鈕要有明確的視覺回饋
- 拖曳、滑動要有流暢動畫
- 完成任務要有正面鼓勵
- 錯誤提示要溫和友善

#### 4. 易於分享 (Shareable)

- 重要資訊要容易截圖分享
- 支援生成美觀的分享卡片
- 每個頁面都可以直接分享連結

## 使用者情境

### 情境 1: 規劃東京 5 日遊

**角色**: 小美（28 歲，行銷專員）  
**目標**: 規劃三月的東京賞櫻之旅

**使用流程**:

1. 📱 打開 Travelogue，建立新旅程「2024 東京賞櫻」
2. 📅 設定日期：3/20 - 3/24
3. 🗺️ 開始新增行程：
   - 看到橫向捲動的日期選擇器
   - 選擇 3/20，新增「淺草寺」
   - 系統自動顯示當天天氣預報
4. 🎫 新增預訂資料：
   - 上傳機票電子檔
   - 看到登機證風格的卡片顯示
5. 💰 設定預算與記帳提醒

**設計需求**:

- 新手引導流程要流暢
- 每個步驟都有視覺回饋
- 可以快速跳過不需要的步驟

---

### 情境 2: 旅行中快速查看資訊

**角色**: 小美（在東京成田機場）  
**目標**: 確認今晚的飯店資訊

**使用流程**:

1. 📱 打開 App（從手機桌面，PWA 已安裝）
2. 👆 點擊底部導航的「預訂」
3. 🔐 輸入 PIN 碼解鎖
4. 🏨 看到今晚住宿卡片
5. 📍 點擊地址，自動開啟 Google Maps 導航

**設計需求**:

- 開啟速度要快（< 2 秒）
- 重要資訊要大而清楚
- 一鍵開啟導航

---

### 情境 3: 餐廳結帳後記帳

**角色**: 小美（在居酒屋結帳後）  
**目標**: 快速記錄這筆花費

**使用流程**:

1. 📱 打開 App，點擊「記帳」
2. ➕ 點擊「新增支出」
3. 💴 輸入金額：3,500 JPY
4. 🍱 選擇類別：美食 > 晚餐
5. 👥 選擇分攤：與男友均分
6. ✅ 系統自動換算成台幣顯示
7. 💾 儲存完成，回到列表看到新紀錄

**設計需求**:

- 輸入流程最多 3 步
- 數字鍵盤要大
- 常用選項預設在前面

---

### 情境 4: 旅行後寫日誌

**角色**: 小美（回家後）  
**目標**: 整理旅行照片與心得

**使用流程**:

1. 📱 打開 App，進入「日誌」
2. ✍️ 新增日誌：「Day 1 - 淺草寺初體驗」
3. 📷 上傳 5 張照片
4. 📝 撰寫文字
5. 😊 選擇心情標記：開心
6. 🔗 關聯到當天行程
7. 👀 預覽美觀的日誌卡片

**設計需求**:

- 照片上傳要簡單（拖曳排序）
- 文字編輯器要易用
- 最終效果要美觀可分享

### 1. 首頁 / 旅程列表

#### 功能需求

- 顯示所有旅程（即將出發 / 進行中 / 已結束）
- 新增旅程按鈕（明顯）
- 每個旅程卡片顯示：封面圖、標題、日期、天數
- 可以左滑刪除或編輯

#### 設計重點

- 封面圖要大且吸引人
- 即將出發的旅程要醒目
- 空狀態要友善引導

### 2. 行程頁面 (Schedule)

#### 功能需求

- **頂部**：倒數計時器 + 天氣卡片
- **日期選擇器**：橫向捲動，顯示整趟旅程的日期
- **時間軸**：當天的所有行程（可拖曳調整順序）
- **每個行程卡片**：
  - 時間
  - 標題
  - 地點
  - 類別圖示（景點/美食/交通/住宿）
  - 拖曳手柄
- **浮動按鈕**：新增行程

#### 設計重點

- 日期選擇器要醒目，選中的日期要放大
- 時間軸左側有虛線連接
- 類別用不同顏色標記
- 拖曳時要有視覺回饋

#### 多選項行程特殊呈現

當同一時段有多個備選方案時：

- 用淺色背景框起來
- 左側標記字母 A、B、C
- 虛線邊框
- 有「選擇其中一個」提示文字

## 關鍵設計需求

### 1. Mobile First 響應式設計

#### 斷點定義

```
Mobile    < 640px   (主要目標)
Tablet    640-1024px
Desktop   > 1024px  (次要支援)
```

#### 設計優先順序

1. **優先設計手機版**（375px - 414px 寬度）
2. 確保平板版可用
3. 桌面版錦上添花

#### 手機版注意事項

- 所有可點擊區域至少 44x44px
- 重要操作要用大拇指可及的範圍
- 底部導航固定不滾動
- 表單輸入要觸發對應鍵盤（數字/文字/Email）

---

### 2. 可分享的視覺設計

每個頁面都要設計得「適合截圖分享」：

- 重要資訊要在一屏內顯示完整
- 配色要美觀和諧
- 可以考慮加上「分享卡片」設計（類似 Notion 的分享預覽）

---

### 3. 動畫與互動

#### 必要動畫

```
頁面切換   Fade + Slide (200ms)
卡片展開   Scale + Fade (250ms)
按鈕點擊   Scale down to 0.95 (150ms)
Loading    骨架屏 + 脈動動畫
成功回饋   ✓ 圖示 + 彈跳動畫
```

#### 拖曳互動

```
行程拖曳   虛線佔位符 + 半透明拖曳元素
Todo拖曳   左側手柄明顯 + 拖曳時放大
```

#### 微互動

```
Checkbox   勾選時✓圖示從小變大
完成項目   文字劃線動畫
刪除操作   滑動顯示刪除按鈕
```

---

### 4. 空狀態設計

每個功能都需要設計「空狀態」：

**原則**

- 用插圖或圖示（不要只有文字）
- 友善引導文案
- 明確的行動呼籲（CTA）

**範例 - 行程空狀態**

```
┌─────────────────────────────┐
│                             │
│         🗺️                  │
│                             │
│     還沒有任何行程           │
│   開始規劃你的旅程吧！       │
│                             │
│    [新增第一個行程]          │
│                             │
└─────────────────────────────┘
```

---

### 5. 錯誤處理與提示

#### 錯誤訊息設計

```
❌ 避免: "Error 404"
✅ 改用: "找不到這個頁面，要回到首頁嗎？"

❌ 避免: "Invalid input"
✅ 改用: "請輸入有效的金額（數字）"
```

#### Toast 通知

```
位置: 頂部中央或底部（不擋住重要內容）
停留: 3 秒自動消失
樣式: 圓角卡片 + 半透明背景 + 圖示
```

---

### 6. 無障礙設計 (Accessibility)

#### 對比度要求

- 一般文字與背景對比度 ≥ 4.5:1
- 大文字（18px+）與背景對比度 ≥ 3:1

#### 點擊區域

- 所有可點擊元素至少 44x44px
- 重要按鈕建議 48x48px 以上

#### 文字可讀性

- 正文字體至少 16px
- 行高至少 1.5 倍字體大小
- 段落間距適當
